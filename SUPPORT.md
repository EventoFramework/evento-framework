# Support

Use the following channels to get help with Evento Framework.

## Documentation

- **Website:** [www.eventoframework.com](https://www.eventoframework.com/)
- **Full docs:** [docs.eventoframework.com](https://docs.eventoframework.com/)
- **Quick start:** [Getting Started guide](https://docs.eventoframework.com/getting-started/quick-start)
- **RECQ patterns:** [eventoframework.com/recq-patterns](https://www.eventoframework.com/recq-patterns/)

## Questions and Discussions

- Start with [README.md](README.md), [CONTRIBUTING.md](CONTRIBUTING.md), and the module READMEs.
- For architecture and developer orientation, read [`CLAUDE.md`](CLAUDE.md) and
  [`.claude/ARCHITECTURE.md`](.claude/ARCHITECTURE.md).
- Open a [GitHub Discussion](https://github.com/EventoFramework/evento-framework/discussions) for
  questions that are not bug reports or feature requests.
- Open a GitHub issue when the question is actionable and repository-specific.

## Bug Reports

Open a bug report using the [GitHub issue template](https://github.com/EventoFramework/evento-framework/issues/new/choose) and include:

- Evento Framework version or commit SHA
- Java version, operating system, and affected module
- Reproduction steps or a minimal failing test
- Expected behavior and actual behavior
- Relevant logs, stack traces, or wire-protocol details

## Security Issues

Do not open public issues for vulnerabilities. Follow [SECURITY.md](SECURITY.md) for the
responsible disclosure process.
